In UNIX systems privileges are assigned to files and directories in form of permissions.
This permissions determina who can read, write or execute file or files in directory.
There are special and regular permissions. Regular ones allow to restrict or grant 
different actions to files / directories to: owner, user group and all other users

There are 3 types of permissions: 1 (execute); 2 (write); 4 (read).
To allow multiply actions just add permission numbers: 5 (read, execute); 3 (write, execute); 1 (execute); 0 (no permissions).

We can separate permissions for different users by setting different permissions for owner, owner groups, others by setting multiuply permission numbers:
- chmod 744 file.txt
The first perm number is for owner
The second perm number is for users of the same grop as owner
The third perm number is for all other users
This command will allow owner to read, write, execute file. And restrict all others only to read it

There also are special permissions:
- chmod 0744 file.txt
Where 0 is special bits.
1 - sticky bit. When set on directory - only file's owner can delete it
2 - SGID (Set Group ID) bit, no matter who runs the file - it will execute with permissions of owner group
4 - SUID (Set User ID) bit, no matter who runs the file - it will run with permissions of it's owner
